# Week 1 — Dataset Engineering Design Spec
**Date:** 2026-07-13
**Project:** E-commerce Return Gatekeeper
**Scope:** Bitext ingestion → DeepSeek-V3 synthetic generation → golden labeling → validation gate → train/val/test split

---

## Decisions Log

| Decision | Choice | Reason |
|---|---|---|
| Execution environment | Local Mac | Week 1 is pure API calls — no GPU needed. Saves RunPod cost. |
| Script structure | Two separate scripts | Generation and labeling are independent; crash in labeling doesn't cost re-running generation (~$0.90). |
| Code architecture | Scripts + shared `lib/` module | DeepSeek client and prompts are reused in Week 3 eval scripts. |
| Concurrency | Async with semaphore (default=15) | 10–20× faster than sequential; respects DeepSeek rate limits. |
| Checkpointing | Yes — append-as-you-go per row | 2,000 API calls; a mid-run crash should not cost re-running from scratch. |
| Multilingual variants | Skipped (English only) | Out of scope for now; can be added later as a separate script. |

## Script 01 — `01_generate_dataset.py`

**Purpose:** Turn 2,000 raw Bitext customer messages into richer, more realistic synthetic dispute messages using DeepSeek-V3.

### Data Flow

1. **Bitext ingestion**
   - Download `bitext/Bitext-customer-support-llm-chatbot-training-dataset` from HuggingFace
   - Filter to 5 target intents: `get_refund`, `cancel_order`, `track_refund`, `complaint`, `check_refund_policy`
   - Sample 400 rows per intent = **2,000 seeds total**
   - Keep only: customer message + intent label
   - Discard: Bitext's original answers (generic, not useful for structured JSON training)

2. **Checkpoint resume**
   - If `data/bitext_seeds.jsonl` already exists, load it and extract already-processed `seed_id`s
   - Skip those rows — only send the delta to DeepSeek

3. **Async generation**
   - Send each unprocessed seed to DeepSeek-V3 using `GENERATION_SYSTEM_PROMPT` (includes ACME policy handbook)
   - Concurrency capped at 15 via asyncio semaphore
   - Each completed row is **immediately appended** to `data/bitext_seeds.jsonl`

4. **Summary**
   - Print: total rows written, rows skipped (already checkpointed), any generation failures

### Output Schema (`data/bitext_seeds.jsonl`)

Each row is a JSON object:
```json
{
  "seed_id": "string — unique ID (e.g. bitext_0042)",
  "original_message": "string — raw Bitext customer message",
  "intent": "string — one of the 5 target intents",
  "synthetic_message": "string — richer dispute message generated by DeepSeek-V3"
}
```

## Script 02 — `02_label_dataset.py`

**Purpose:** For each synthetic dispute message, ask DeepSeek-V3 to produce a structured JSON arbitration decision. Validate every response, discard failures, enforce a ≥1,500 row quality gate, then split into train/val/test sets.

### Data Flow

1. **Load seeds**
   - Read `data/bitext_seeds.jsonl` (requires script 01 to have completed)

2. **Checkpoint resume**
   - If `data/distilled_dataset.jsonl` already exists, load it and skip already-labeled `seed_id`s

3. **Async labeling**
   - Send each seed's `synthetic_message` to DeepSeek-V3 using `LABELING_SYSTEM_PROMPT`
   - Concurrency capped at 15
   - For each response:
     - Validate via `validator.validate()` (JSON parse + REQUIRED_KEYS check)
     - If invalid: retry up to **2 times** with a stricter prompt variant
     - If still invalid after 2 retries: log as discarded, move on
     - If valid: immediately append to `data/distilled_dataset.jsonl`

4. **Quality gate**
   - Assert total valid rows ≥ 1,500
   - If below threshold: exit with clear error showing valid count and discard count

5. **Train/val/test split**
   - Shuffle with fixed random seed (reproducibility)
   - Write:
     - `data/train.jsonl` — 1,200 rows
     - `data/val.jsonl` — 150 rows
     - `data/test.jsonl` — 150 rows (**held out — never used during training**)

### Output Schema (`data/distilled_dataset.jsonl`)

Each row is a JSON object matching the golden label schema from `plan.md`:
```json
{
  "seed_id": "string",
  "synthetic_message": "string",
  "chain_of_thought": "string",
  "intent_action": "get_refund|cancel_order|track_refund|complaint|check_refund_policy",
  "extracted_slots": {
    "order_id": "string|null",
    "invoice_id": "string|null",
    "return_window_days": "int|null",
    "item_condition": "string|null"
  },
  "policy_evaluation": {
    "within_return_window": "bool",
    "item_opened": "bool",
    "evidence_required": "bool"
  },
  "gatekeeper_status": "APPROVE_AUTOMATED|REQUEST_EVIDENCE|ESCALATE_TO_HUMAN",
  "confidence_score": "float",
  "fallback_escalation": "bool",
  "user_facing_response": "string",
  "processing_timestamp": "ISO8601 string"
}
```

## `lib/` Modules

### `deepseek_client.py`
- Wraps `openai.AsyncOpenAI` pointed at DeepSeek's API endpoint
- Holds an `asyncio.Semaphore` (configurable, default=15)
- `generate(user_prompt, system_prompt, max_tokens, temperature) -> str`
- On `429 rate-limit`: exponential backoff, retry up to 3 times before raising
- API key loaded from env var `DEEPSEEK_API_KEY`

### `prompts.py`
- `POLICY_HANDBOOK` — ACME Corp returns policy text (used in both scripts)
- `GENERATION_SYSTEM_PROMPT` — for script 01; instructs DeepSeek to produce a richer dispute message
- `LABELING_SYSTEM_PROMPT` — for script 02; instructs DeepSeek to produce a structured JSON decision
- `LABELING_SYSTEM_PROMPT_STRICT` — stricter variant used on retry attempts
- No logic — strings only

### `checkpoint.py`
- `load_checkpoint(path: str) -> list[dict]` — reads `.jsonl`, returns list of row dicts; returns `[]` if file doesn't exist
- `append_row(path: str, row: dict) -> None` — appends one JSON line to file; thread/coroutine safe

### `validator.py`
- `REQUIRED_KEYS: list[str]` — the 6 required fields from the golden label schema
- `validate(response_str: str) -> tuple[bool, dict | None]` — parse JSON + check keys; returns `(True, parsed)` or `(False, None)`

## Error Handling Summary

| Scenario | Behaviour |
|---|---|
| DeepSeek 429 rate limit | Exponential backoff, retry up to 3× |
| Invalid JSON response | Retry up to 2× with stricter prompt; discard if still fails |
| Script crash mid-run | Resume from last checkpointed row on next run |
| <1,500 valid rows after labeling | Script exits with error; shows valid/discard counts |
| Missing `.env` / API key | Script exits immediately with clear message |
